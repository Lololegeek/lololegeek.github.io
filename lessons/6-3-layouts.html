Layouts Flexibles

Apprenez à organiser vos éléments automatiquement sans calculer manuellement chaque position.

Roblox Studio Requis

Cette leçon utilise l'API de Roblox. Ce code ne fonctionnera que dans Roblox Studio ou notre éditeur intégré, et non sur un interpréteur Lua PC standard.

Pourquoi utiliser des Layouts ?

Calculer la position exacte de 10 boutons dans un menu est fastidieux. Si vous ajoutez un 11ème bouton, vous devez tout recalculer. Les UI Layouts font ce travail pour vous !

UIListLayout

C'est le plus utilisé. Il aligne tous les enfants d'une Frame en liste (verticale ou horizontale).

Propriétés utiles :

  • FillDirection : Vertical (haut-bas) ou Horizontal (gauche-droite).
  • Padding : L'espace entre chaque élément (Scale ou Offset).
  • SortOrder : L'ordre d'affichage (ex: par Nom ou par l'ordre d'ajout).

UIGridLayout

Idéal pour les inventaires ou les boutiques. Il organise les éléments dans une grille avec des colonnes et des lignes.

Contrairement au ListLayout, le GridLayout impose sa propre taille (CellSize) à tous ses éléments enfants.

Les Constraints (Contraintes)

Pour que votre UI reste parfaite sur tous les écrans, utilisez ces aides :

UIAspectRatioConstraint

Force un élément à garder ses proportions (ex: toujours un carré) peu importe la taille de l'écran.

UIPadding

Ajoute des marges intérieures à un conteneur pour que les éléments ne collent pas aux bords.

UISizeConstraint

Définit une taille minimum et maximum pour vos fenêtres.
