Les Services

Les Services sont les composants principaux de Roblox qui gèrent des fonctionnalités spécifiques : joueurs, stockage, interface, etc.

Roblox Studio Requis

Cette leçon utilise l'API de Roblox. Ce code ne fonctionnera que dans Roblox Studio ou notre éditeur intégré, et non sur un interpréteur Lua PC standard.

Qu'est-ce qu'un Service ?

Les Services sont des objets spéciaux qui gèrent différentes parties du jeu. Ils sont uniques (il n'y en a qu'un de chaque type) et accessibles partout.

Players

Gère les joueurs connectés

ReplicatedStorage

Stockage partagé client/serveur

ServerStorage

Stockage serveur uniquement

Workspace

Contient tout le monde 3D

Accéder aux Services

services.lua 
-- Méthode 1 : GetService (RECOMMANDÉE)
local Players = game:GetService("Players")
local RS = game:GetService("ReplicatedStorage")
local SS = game:GetService("ServerStorage")

-- Méthode 2 : Accès direct (moins flexible)
local players = game.Players
local workspace = game.Workspace

-- Services utiles les plus communs
local TweenService = game:GetService("TweenService")
local RunService = game:GetService("RunService")
local UserInputService = game:GetService("UserInputService")

Pourquoi GetService ?

GetService() est plus fiable car il fonctionne même si le service n'apparaît pas encore dans l'Explorer, et donne une meilleure autocomplétion dans l'éditeur.

Le Service Players

players_service.lua 
local Players = game:GetService("Players")

-- Obtenir tous les joueurs connectés
local allPlayers = Players:GetPlayers()
print("Joueurs en ligne : " .. #allPlayers)

-- Obtenir le joueur local (SEULEMENT côté client)
local localPlayer = Players.LocalPlayer
print("Bienvenue " .. localPlayer.Name)

-- Écouter quand un joueur rejoint
Players.PlayerAdded:Connect(function(player)
    print(player.Name .. " a rejoint le jeu !")
end)

-- Écouter quand un joueur quitte
Players.PlayerRemoving:Connect(function(player)
    print(player.Name .. " a quitté le jeu")
end)

Stockages

storages.lua 
-- ReplicatedStorage : objets partagés entre client et serveur
local RS = game:GetService("ReplicatedStorage")
local epee = RS:WaitForChild("Epee"):Clone()
epee.Parent = player.Backpack

-- ServerStorage : objets SEULEMENT accessibles côté serveur
-- Parfait pour les modèles de mobs, trésors secrets, etc.
local SS = game:GetService("ServerStorage")
local boss = SS.Mobs.BossFinal:Clone()
boss.Parent = workspace

Client vs Serveur

ServerStorage n'est accessible que depuis les scripts serveur. Essayer d'y accéder depuis un LocalScript provoquera une erreur !

Résumé

  • Utilisez game:GetService("NomDuService")
  • Players : gestion des joueurs
  • Workspace : le monde 3D
  • ReplicatedStorage : partagé client/serveur
  • ServerStorage : serveur uniquement
  • TweenService : animations fluides
