Les Dictionnaires

Apprenez à utiliser les tables comme dictionnaires pour associer des clés à des valeurs et organiser vos données de manière plus lisible.

Qu'est-ce qu'un Dictionnaire ?

Un dictionnaire est une table qui utilise des clés nommées plutôt que des indices numériques. On l'appelle aussi "table associative" ou "hash map".

C'est parfait pour représenter des objets avec des propriétés, comme un joueur avec son nom, niveau, et points.

Créer un Dictionnaire

dictionnaire.lua 
-- Créer un dictionnaire
local joueur = {
    nom = "Alex",
    niveau = 25,
    points = 1500,
    estVIP = true
}

-- Deux façons d'accéder aux valeurs :

-- 1. Notation point (la plus courante)
print(joueur.nom)     -- "Alex"
print(joueur.niveau)  -- 25

-- 2. Notation crochet (utile pour les clés dynamiques)
print(joueur["nom"])     -- "Alex"
print(joueur["points"])  -- 1500

Quand utiliser les crochets ?

Utilisez table["clé"] quand la clé est stockée dans une variable ou contient des caractères spéciaux. Sinon, préférez table.clé qui est plus lisible.

Modifier un Dictionnaire

modification.lua 
local voiture = {
    marque = "Ferrari",
    couleur = "Rouge",
    vitesse = 300
}

-- Modifier une valeur existante
voiture.vitesse = 350
print(voiture.vitesse)  -- 350

-- Ajouter une nouvelle clé
voiture.annee = 2024
print(voiture.annee)    -- 2024

-- Supprimer une clé (mettre à nil)
voiture.couleur = nil
print(voiture.couleur)  -- nil

L'opérateur # ne fonctionne pas !

#dictionnaire retourne 0 pour les dictionnaires car il ne compte que les indices numériques séquentiels. Pour compter les clés d'un dictionnaire, il faut boucler dessus.

Exemple Pratique : Données Joueur

player_data.lua 
-- Structure de données d'un joueur Roblox
local playerData = {
    userId = 123456789,
    username = "CoolPlayer2024",
    
    -- Dictionnaire imbriqué pour les statistiques
    stats = {
        coins = 5000,
        gems = 150,
        level = 42
    },
    
    -- Tableau dans le dictionnaire pour l'inventaire
    inventory = {"Épée", "Bouclier", "Potion"}
}

-- Accéder aux données imbriquées
print(playerData.stats.coins)    -- 5000
print(playerData.inventory[1])   -- "Épée"

-- Modifier les stats
playerData.stats.coins = playerData.stats.coins + 100
print(playerData.stats.coins)    -- 5100

Clés Dynamiques

cles_dynamiques.lua 
local scores = {}

-- Ajouter des scores avec des noms de joueurs comme clés
local nomJoueur = "Alex"
scores[nomJoueur] = 500  -- Équivalent à scores["Alex"] = 500

scores["Bob"] = 750
scores["Charlie"] = 300

-- Vérifier si une clé existe
if scores["Alex"] then
    print("Alex a un score de : " .. scores["Alex"])
end

-- Accéder à une clé inexistante retourne nil
print(scores["David"])  -- nil

Résumé

  • Les dictionnaires utilisent des clés nommées au lieu d'indices numériques
  • Deux syntaxes : table.clé ou table["clé"]
  • Pour supprimer une clé : table.clé = nil
  • #table ne fonctionne PAS pour les dictionnaires
  • On peut imbriquer des dictionnaires et des tableaux
  • Accéder à une clé inexistante retourne nil
