Obby Basique : Killbricks & Checkpoints

Les Obbys (parcours d'obstacles) sont les jeux les plus populaires sur Roblox. Apprenez à programmer les éléments essentiels : la lave qui tue et les points de sauvegarde.

Roblox Studio Requis

Cette leçon utilise l'API de Roblox. Ce code ne fonctionnera que dans Roblox Studio ou notre éditeur intégré, et non sur un interpréteur Lua PC standard.

Créer une KillBrick (Lave)

Une KillBrick est une brique qui tue le joueur au contact. Pour cela, nous utilisons l'événement Touched.

killbrick.lua 
local lave = script.Parent

local function tuerJoueur(hit)
    -- Vérifier si c'est un personnage qui a touché
    local humanoid = hit.Parent:FindFirstChild("Humanoid")
    
    if humanoid then
        -- Mettre la santé à 0 = Mort
        humanoid.Health = 0
    end
end

lave.Touched:Connect(tuerJoueur)

Attention au Debounce

L'événement Touched se déclenche très vite et plusieurs fois. Pour tuer, ce n'est pas grave, mais pour donner des récompenses, il faudra un système anti-spam (debounce).

Système de Checkpoints

Roblox gère les Spawns automatiquement grâce aux SpawnLocation et leur propriété TeamColor, mais pour un Obby linéaire, nous voulons changer le point de réapparition quand on touche un checkpoint.

1. Utiliser les SpawnLocation

La méthode la plus simple sans script complexe :

  • Ajoutez des objets SpawnLocation
  • Cochez AllowTeamChangeOnTouch
  • Cochez Neutral = false
  • Donnez une TeamColor unique à chaque checkpoint
  • Créez les Teams correspondantes dans le service Teams

2. Méthode Scriptée (Plus pro)

Pour éviter d'avoir 50 Teams, on utilise les Leaderstats pour stocker le niveau du joueur.

checkpoint_manager.lua 
local Players = game:GetService("Players")

-- Quand un joueur rejoint
Players.PlayerAdded:Connect(function(player)
    -- Créer leaderstats avec Stage
    local stats = Instance.new("Folder")
    stats.Name = "leaderstats"
    stats.Parent = player
    
    local stage = Instance.new("IntValue")
    stage.Name = "Stage"
    stage.Value = 1
    stage.Parent = stats
    
    -- Gérer la réapparition
    player.CharacterAdded:Connect(function(character)
        -- Attendre un peu que le personnage charge
        wait(0.1)
        
        -- Trouver le checkpoint correspondant
        local checkpoint = workspace.Checkpoints:FindFirstChild(stage.Value)
        
        if checkpoint then
            -- Téléporter le joueur
            character:MoveTo(checkpoint.Position)
        end
    end)
end)

Script du Checkpoint

Ce script va dans chaque brique de checkpoint (nommées "1", "2", "3"... dans un dossier "Checkpoints").

checkpoint_touch.lua 
local checkpoint = script.Parent
local numeroStage = tonumber(checkpoint.Name) -- Le nom doit être "1", "2"...

checkpoint.Touched:Connect(function(hit)
    local player = game.Players:GetPlayerFromCharacter(hit.Parent)
    
    if player then
        local st = player.leaderstats.Stage
        
        -- Si le joueur atteint le checkpoint suivant
        if st.Value == numeroStage - 1 then
            st.Value = numeroStage
            
            -- Effet visuel (changement de couleur vert)
            checkpoint.BrickColor = BrickColor.new("Lime green")
        end
    end
end)

Résumé

  • KillBrick : Utilise Touched et met Humanoid.Health = 0
  • Checkpoints : Utilisent une valeur "Stage" dans les leaderstats
  • CharacterAdded : Event qui se lance quand le joueur respawn
  • MoveTo : Fonction pour téléporter un Model (le personnage)
