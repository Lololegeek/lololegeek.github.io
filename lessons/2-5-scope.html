Scope (Portée) & Closures

Comprendre où vos variables sont visibles et combien de temps elles existent est essentiel pour éviter les bugs.

Qu'est-ce que le Scope ?

Le scope (portée) définit où une variable est visible. En Lua, le scope est déterminé par les blocs do ... end, if ... end, function ... end, etc.

scope.lua 
local variableGlobaleAuScript = 10

if true then
    local variableLocaleAuBloc = 20
    print(variableGlobaleAuScript) -- Possible : 10
    print(variableLocaleAuBloc)    -- Possible : 20
end

-- print(variableLocalAuBloc) -- ERREUR ! Elle n'existe plus ici (nil)

Variables Globales vs Locales

Une variable déclarée sans local est globale (à tout le jeu !). C'est généralement une mauvaise pratique.

Pourquoi éviter les global ?

1. Plus lent à accéder
2. Risque de conflits avec d'autres scripts
3. Plus dur à debugger

Upvalues & Closures

Une fonction créée "à l'intérieur" d'une autre peut se "souvenir" des variables locales même après la fin de la fonction parente. C'est magique !

closure.lua 
local function creerCompteur()
    local compte = 0
    
    return function()
        compte = compte + 1
        return compte
    end
end

local monCompteur = creerCompteur()

print(monCompteur()) -- 1
print(monCompteur()) -- 2
print(monCompteur()) -- 3

Exercice Pratique

Jouez avec les portées de variables !

  1. Créez une variable globale score
  2. Créez un bloc do ... end
  3. À l'intérieur du bloc, créez une variable local score = 10 (attention au nom !)
  4. Affichez le score à l'intérieur et à l'extérieur du bloc.

Résumé

  • Utilisez toujours local
  • Les variables locales meurent à la fin de leur bloc end
  • Les fonctions peuvent "capturer" des variables externes (closures)
