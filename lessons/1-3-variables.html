Les Variables

Les variables sont comme des boîtes qui stockent des données. Apprenez à les créer et les utiliser efficacement.

Qu'est-ce qu'une variable ?

Une variable est un espace de stockage avec un nom. Imaginez-la comme une boîte étiquetée où vous pouvez mettre une valeur.

Stocker

Garde une valeur en mémoire

Modifier

Change la valeur à tout moment

Réutiliser

Utilise la valeur plusieurs fois

Nommer

Donne un nom explicite

Créer une variable

En Lua, on crée une variable avec le mot-clé local :

variables.lua 
-- Syntaxe: local nomVariable = valeur

local nom = "Alex"
local age = 16
local estConnecte = true
local score = 0

-- Afficher les variables
print(nom)        -- Alex
print(age)         -- 16
print(estConnecte) -- true

Toujours utiliser local

Utilisez TOUJOURS local pour créer vos variables. Les variables sans local sont globales et peuvent causer des problèmes.

Modifier une variable

Vous pouvez changer la valeur d'une variable à tout moment :

modifier.lua 
local score = 0
print(score)  -- 0

score = 100  -- Pas de 'local' pour modifier !
print(score)  -- 100

score = score + 50  -- Ajouter 50
print(score)  -- 150

score = score * 2   -- Doubler
print(score)  -- 300

Astuce

score = score + 1 signifie "prends la valeur actuelle de score, ajoute 1, et stocke le résultat dans score".

Règles de nommage

Les noms de variables doivent suivre ces règles :

  • Commencer par une lettre ou un underscore _
  • Contenir des lettres, chiffres et underscores
  • Ne pas être un mot réservé (if, then, end, etc.)
  • Sensible à la casse (Score ≠ score)
nommage.lua 
-- ✅ Noms valides
local nom = "Alex"
local monScore = 100
local player1 = "Bob"
local _private = true
local NomDuJoueur = "Max"

-- ❌ Noms invalides
-- local 1player = "Nope"  -- Commence par un chiffre
-- local mon-score = 100   -- Contient un tiret
-- local if = "test"       -- Mot réservé

Conventions de style

En Lua/Roblox, on utilise généralement le camelCase :

  • playerName au lieu de player_name
  • maxHealth au lieu de MaxHealth
  • isReady au lieu de is_ready

Déclarations multiples

Vous pouvez déclarer plusieurs variables en une ligne :

multiples.lua 
-- Une variable par ligne (recommandé)
local x = 10
local y = 20
local z = 30

-- Plusieurs sur une ligne (possible mais moins lisible)
local a, b, c = 1, 2, 3

print(a, b, c)  -- 1  2  3

-- Échanger deux valeurs facilement !
a, b = b, a
print(a, b)  -- 2  1

Exercice

Créez un programme qui :

  1. Crée une variable pour la vie d'un joueur (100)
  2. Crée une variable pour les dégâts (25)
  3. Enlève les dégâts de la vie
  4. Affiche la vie restante
Ouvrir l'Éditeur

Résumé

  • Utilisez local pour créer des variables
  • Les variables stockent des valeurs que vous pouvez modifier
  • Choisissez des noms explicites en camelCase
  • Ne pas utiliser de mots réservés ou commencer par un chiffre
