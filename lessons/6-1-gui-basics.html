Introduction aux Interfaces (GUI)

Apprenez à créer des éléments visuels qui s'affichent sur l'écran du joueur (HUD, menus, inventaires).

La Hiérarchie du GUI

Dans Roblox, toutes les interfaces 2D sont stockées dans le service StarterGui. Voici la structure typique :

  1. StarterGui (Le conteneur global)
  2. ScreenGui (Le "calque" qui couvre l'écran)
  3. Frame (Un rectangle/panneau pour organiser les éléments)
  4. Éléments (Boutons, Textes, Images)

ScreenGui vs Frame

Le ScreenGui est indispensable. Sans lui, rien ne s'affiche à l'écran. Il possède une propriété importante : ResetOnSpawn. Si elle est cochée, l'interface se réinitialise quand le joueur meurt.

La Frame est comme un dossier visuel. Elle permet de regrouper plusieurs boutons ou textes pour les déplacer ou les masquer tous en même temps.

Position et Taille (Scale vs Offset)

C'est l'erreur n°1 des débutants. Roblox utilise deux systèmes :

  • Scale (0 à 1) : Pourcentage de l'écran. {0.5, 0} = Pile au milieu. À favoriser !
  • Offset (Pixels) : Taille fixe. {0, 100} = 100 pixels. Attention : sur mobile, 100px c'est énorme !
Utilise toujours le Scale pour que ton UI soit "Responsive" (adaptée à tous les écrans).
