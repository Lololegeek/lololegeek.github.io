Les Events

Les Events permettent à votre code de réagir aux actions : clics, collisions, joueurs qui rejoignent... Maîtrisez la programmation événementielle !

Qu'est-ce qu'un Event ?

Un Event est un signal qui se déclenche quand quelque chose se passe dans le jeu. Au lieu de vérifier constamment si quelque chose s'est passé, on "écoute" l'événement et on réagit.

Programmation événementielle

C'est comme une sonnette : au lieu de regarder par la fenêtre toutes les 5 secondes pour voir si quelqu'un arrive, vous attendez que la sonnette sonne !

Connecter un Event

connect.lua 
-- Syntaxe de base : Event:Connect(fonction)

local part = workspace.MaPart

-- Méthode 1 : Fonction anonyme
part.Touched:Connect(function(autrePart)
    print("Touché par : " .. autrePart.Name)
end)

-- Méthode 2 : Fonction nommée
local function onTouched(autrePart)
    print("Collision avec " .. autrePart.Name)
end

part.Touched:Connect(onTouched)

Events les Plus Utilisés

events_communs.lua 
local Players = game:GetService("Players")

-- 1. PlayerAdded - Un joueur rejoint
Players.PlayerAdded:Connect(function(player)
    print(player.Name .. " a rejoint !")
end)

-- 2. Touched - Collision avec une part
workspace.Lava.Touched:Connect(function(hit)
    local humanoid = hit.Parent:FindFirstChild("Humanoid")
    if humanoid then
        humanoid.Health = 0  -- Mort instantanée
    end
end)

-- 3. MouseButton1Click - Clic sur un bouton GUI
button.MouseButton1Click:Connect(function()
    print("Bouton cliqué !")
end)

-- 4. ChildAdded - Un enfant est ajouté
workspace.ChildAdded:Connect(function(child)
    print("Nouvel objet : " .. child.Name)
end)

Déconnecter un Event

disconnect.lua 
-- Stocker la connexion
local connexion

connexion = part.Touched:Connect(function(hit)
    print("Touché une fois !")
    
    -- Déconnecter après le premier touch
    connexion:Disconnect()
end)

-- Ou utiliser :Once() pour un seul déclenchement
part.Touched:Once(function(hit)
    print("Ceci ne s'exécute qu'une seule fois !")
end)

Évitez les fuites de mémoire

Si vous créez des connexions dans une boucle ou pour des objets temporaires, pensez à les déconnecter quand vous n'en avez plus besoin !

ClickDetector

click_detector.lua 
-- Script dans une Part avec un ClickDetector
local part = script.Parent
local clickDetector = part:FindFirstChildOfClass("ClickDetector")

clickDetector.MouseClick:Connect(function(player)
    print(player.Name .. " a cliqué sur la part !")
    
    -- Changer la couleur au clic
    part.BrickColor = BrickColor.Random()
end)

-- Hover (souris au-dessus)
clickDetector.MouseHoverEnter:Connect(function(player)
    part.Material = Enum.Material.Neon
end)

clickDetector.MouseHoverLeave:Connect(function(player)
    part.Material = Enum.Material.Plastic
end)

Résumé

  • Event:Connect(fonction) écoute un événement
  • Event:Once(fonction) n'écoute qu'une seule fois
  • connexion:Disconnect() arrête d'écouter
  • Touched : collision physique
  • PlayerAdded/Removing : joueurs entrent/sortent
  • MouseClick : clic via ClickDetector
