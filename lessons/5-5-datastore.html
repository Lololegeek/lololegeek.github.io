DataStore : Sauvegardes

Le sujet le plus important : sauvegarder la progression des joueurs pour qu'ils ne perdent pas leurs niveaux quand ils quittent le jeu.

Activer les Services API

Avant de coder, vous devez activer l'accès aux DataStores dans Roblox Studio :

  • Allez dans Game Settings (onglet Home)
  • Allez dans l'onglet Security
  • Activez "Enable Studio Access to API Services"

Script de Sauvegarde Complet

Ce script gère le chargement à la connexion et la sauvegarde à la déconnexion.

datastore_system.lua 
local Players = game:GetService("Players")
local DataStoreService = game:GetService("DataStoreService")

-- Créer ou récupérer le DataStore "PlayerData"
local myStore = DataStoreService:GetDataStore("PlayerData_v1")

-- 1. CHARGEMENT DES DONNÉES
Players.PlayerAdded:Connect(function(player)
    -- Créer les leaderstats (voir leçon 5.1)
    local leaderstats = Instance.new("Folder")
    leaderstats.Name = "leaderstats"
    leaderstats.Parent = player
    
    local coins = Instance.new("IntValue")
    coins.Name = "Coins"
    coins.Parent = leaderstats
    
    -- Clé unique pour le joueur (ex: "Player_12345")
    local key = "Player_" .. player.UserId
    
    -- Essayer de charger les données
    local success, data = pcall(function()
        return myStore:GetAsync(key)
    end)
    
    if success then
        -- Si data existe, on l'utilise, sinon 0
        coins.Value = data or 0
        print("Données chargées pour " .. player.Name)
    else
        warn("Erreur de chargement !")
    end
end)

-- 2. SAUVEGARDE DES DONNÉES
local function saveData(player)
    local leaderstats = player:FindFirstChild("leaderstats")
    if leaderstats then
        local key = "Player_" .. player.UserId
        local coinsToSave = leaderstats.Coins.Value
        
        -- Essayer de sauvegarder
        local success, errorMsg = pcall(function()
            myStore:SetAsync(key, coinsToSave)
        end)
        
        if success then
            print("Données sauvegardées !")
        else
            warn("Erreur de sauvegarde: " .. errorMsg)
        end
    end
end

-- Sauvegarder quand un joueur part
Players.PlayerRemoving:Connect(saveData)

-- Sauvegarder TOUT LE MONDE quand le serveur ferme (Important !)
game:BindToClose(function()
    for _, player in ipairs(Players:GetPlayers()) do
        saveData(player)
    end
end)

pcall (Protected Call)

Les appels au DataStore peuvent échouer (problème internet, serveur Roblox...). pcall permet d'attraper l'erreur sans faire planter tout le script.

Résumé

  • Activer "Enable Studio Access to API Services"
  • Utiliser DataStoreService:GetDataStore("Nom")
  • GetAsync(key) pour charger
  • SetAsync(key, value) pour sauvegarder
  • Toujours utiliser pcall pour gérer les erreurs
  • Utiliser BindToClose pour sauvegarder à la fermeture du serveur
